Brand Custom Request Use Case Document
Use Case 15: Brand Custom Nail Request Scenarios
Introduction
This document provides a comprehensive overview of Use Case 15: Brand Custom Request for nail services. It details thirteen distinct scenarios encompassing how a brand company submits custom nail requests, the flows for client and artist selection, request status changes, and interface elements supporting these workflows. The intended audience includes product managers, developers, and QA teams, ensuring clarity and actionable guidance for implementation.
Purpose and Scope
The purpose of this document is to define the operational and technical requirements for Brand Custom Request scenarios, focusing on the interactions between brand companies, clients, and artists. The scope includes request submission, selection logic, workflow states, user actions (Accept/Cancel), and order status updates.
Actors
Brand Company: Submits custom nail requests and selects client and artist pools or specific entities.
Client: Receives requests, reviews details, accepts or declines, and interacts via provided interface elements.
Artist: Reviews accepted requests from clients or brand companies, accepts or declines assignments, and updates order status.

Status and Order Updates
Pending: Request submitted, awaiting client action.
In Review: Client accepted, artist review in progress.
Declined: Client or artist declined the request.
Expired: No action within defined timeframe; request automatically expires.
Cancelled: Request canceled by user or brand company.
Status transitions are triggered by user actions (Accept, Decline, Cancel), system expiry, or cancellation events.
Interface Elements
Accept Button: Allows client or artist to accept the request.
Cancel Button: Enables user to cancel the request.
Dropdowns: For selecting clients, artists, or groups during request submission.
Request Detail Page: Displays request information, status, and action buttons for all parties.
